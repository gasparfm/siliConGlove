Glove as connection manager

This example is using Glove to accept connections (also HTTPs connections can be accepted). Each Virtualhost can have several routes configured and each route have a callback. This callback is used to generate a response. This response includes HTTP headers to send, and the HTML content (or image, or CSS).

With Glove, you can setup routes with keywords so all /docs/$document will end up in the same callback function, and this function will receive the value of $document to modify it's behaviour.

Silicon con manage templates

One of the worse tasks when coding web sites in C++ is compiling everytime you make a tiny change. You can avoid this 100% but using a template system may help a little bit.

Of course it adds a bit overhead, but worth it. The template system can use simple conditionals and loops (under development), functions and simple keywords.

Keywords are simple replacements in your document, functions may call C++ functions (configurable) to generate the response; conditions and loops may save you some time when generating complex responses
